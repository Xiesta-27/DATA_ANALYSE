## 天猫订单数据分析项目 tmall_order_analysis

## 一、项目背景
随着电商业务规模扩大，订单数据成为反映市场分布、用户行为、业务健康度的核心依据。本项目通过 MySQL 做数据清洗与分析，结合 Jupyter + pyecharts + matplotlib 实现可视化，完成从原始数据到业务洞察的全流程分析，为市场运营、供应链优化、活动规划提供数据支撑。

## 二、技术栈
- 数据库：MySQL
- 数据分析：SQL
- 可视化工具：pyecharts、matplotlib
- 开发环境：Jupyter Notebook
- 数据分析库：pandas


## 三、分析内容
### 1. 数据预处理
- 创建数据库与订单表。
- 导入 CSV 原始数据。
- 清洗重复数据。
- 省份字段标准化（去除省 / 自治区 / 特别行政区等后缀）。

### 2. 整体运营指标分析
- 总订单数量、已完成订单数、未付款订单数。
- 总订单金额、总退款金额、总实际收入。
- 成交率、退货率。
  
### 3. 地区订单分布分析
- 统计各省份订单量。
- 绘制全国订单分布地图。
- 识别核心市场与薄弱区域。

### 4. 时间趋势分析
- 每日订单量走势。
- 每小时订单量分布。
- 业务周期波动规律识别。

### 5. 转化效率分析
- 下单到付款平均耗时分析。


## 四、可视化结果分析
- 全国订单分布结论
从可视化地图可看出：
核心优势市场：上海、广东、江苏、浙江、北京等东部沿海省市订单量遥遥领先，是主要营收来源。
业务短板区域：西藏、青海、宁夏、新疆等西部偏远地区订单量极少，属于市场覆盖盲区。
验证结论：一二线城市、人口密集、高消费能力地区是电商主力市场。
![全国订单分布](pyecharts%20picture/全国订单分布图.png)

- 每日订单量趋势结论
2 月份每日订单呈现明显三段式周期：
业务低谷期（2.10–2.15）：几乎无订单，受春节假期 + 疫情初期影响。
V 型复苏期（2.16–2.22）：订单快速上涨，复工复产 + 线上消费需求爆发。
峰值震荡期（2.24 之后）：达到峰值 3416 单，回落至 2000–2800 单区间，为真实业务承载力。
![每日订单走势](pyecharts%20picture/每日订单走势图.png)

## 五、业务优化建议
市场运营：深耕东部核心市场，加大投放与会员运营；重点培育四川、河南、湖北等潜力省份。
供应链优化：在华东、华南、华北布局核心仓库，匹配订单密集区，降低物流成本。
活动策略：不将短期脉冲峰值当作长期趋势，避免过度备货；通过 A/B 测试优化活动节奏。
风险预案：建立节假日 / 特殊事件应对机制，降低外部环境对业务的冲击。
